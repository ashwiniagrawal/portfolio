Case studies

Case study 01 — Enterprise AI platform

Agentic Platform — Memory Engineering console

A control-plane screen for platform admins to govern what AI agents and domain packs are allowed to remember, retrieve, retain, share, and delete. The challenge was making a genuinely technical, permission-heavy system — profiles, approvals, quarantine states, runtime evidence — legible to admins without flattening the nuance they need to trust it.

Governance an admin can audit at a glance Dense state (active / draft / pending / quarantined) made scannable An in-context AI advisor for configuration, not a bolted-on chatbot
Enterprise UXInformation architectureAdmin console design

Case study 02 — Restaurant operations SaaS

General Manager Virtual Agent

A dashboard built for restaurant GMs running a shift on the floor — live menu availability, a mini kitchen display system tracking orders against SLA, and a conversational AI assistant sitting alongside it. The design pairs a chat-first interface with a structured visual layer, so a GM can ask a question or just glance at the numbers, whichever the moment calls for.

Status, stock, and SLA readable in one glance mid-shift Chat and dashboard as one system, not two separate tools Designed for tablet use in a fast-moving kitchen environment

Case study 03 — Data operations & analytics

Data validation configuration + pipeline analytics

Two ends of the same data operations workflow: a setup screen where technical and non-technical users configure source-to-target validation rules without writing a script, and a revenue analytics dashboard that turns that underlying data into a pipeline view executives actually read. Same design system, two very different audiences.

Configuration steps written in plain language, not database terms One visual language spanning setup screens and executive dashboards Dense pipeline data organized around the decisions it supports

How I work

Process

Discover

Understand the workflow

Interviews with the people actually using the tool daily — support, ops, finance — not just stakeholders.

Structure

Map the data model

Information architecture and permissions first; the UI follows once the underlying model is right.

Design

Prototype in context

Real data, real edge cases, real density — dashboards fail in the empty states and the 200-row tables.

Validate

Test with power users

Usability sessions with people who'll use this five times a day, not five times a year.

About

A little more about me

Ashwini Agrawal is a Senior Frontend Engineer and product designer with 8+ years building scalable enterprise web applications — from complete UI design through to production React and Angular builds. She specializes in high-performance UI platforms, complex data visualization, and AI-integrated enterprise tools, and has shipped everything from Azure AD-secured analytics platforms to conversational AI agent interfaces.

Currently a Designer Analyst at Bitwise India, where she owns UI design end-to-end and builds it herself — pairing Figma with React, Angular, and TypeScript so nothing gets lost in handoff. She's drawn to messy, data-dense enterprise problems: hierarchical pipelines, permission systems, large-file editors — the interfaces most designers avoid.
